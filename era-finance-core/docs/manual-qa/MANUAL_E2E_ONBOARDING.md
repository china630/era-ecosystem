# Ручные E2E-сценарии: онбординг компании

Документ для **QA и приёмки**: предусловия, шаги, ожидаемый результат. Порядок разделов соответствует **типичному запуску новой организации** в ERA Finance.

**Источники:** [PRD.md](../../PRD.md) §3–§4, [TZ.md](../../TZ.md) §0.0, §2, §6.0, §7, §10–§11; навигация по коду — `.cursor/rules/erafinance-module-map.mdc`.

**Не дублирует** пошаговые объяснения UI: для бухгалтера см. [manual-buhgalter.md](../manual-accountant/manual-buhgalter.md).

---

## Шаблон сценария

| Поле | Описание |
|------|----------|
| **ID** | Уникальный идентификатор кейса |
| **Модуль** | Ссылка на PRD §4.x / TZ |
| **Роли** | Минимально достаточные роли (`OWNER`, `ADMIN`, `ACCOUNTANT`, …) |
| **Предусловия** | Данные, модули тарифа, окружение |

| **Ожидаемый результат** | Видимое состояние, сообщения, данные в отчётах |
| **Негатив** | Опционально: ошибка, блокировка, код ответа API |

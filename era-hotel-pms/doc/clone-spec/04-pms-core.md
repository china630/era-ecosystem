## 4.1 Сущности и связи

```
Гость 1──N Бронь 1──0..1 Проживание 1──1 Folio
                │              │
                └── Тип номера └── Физ. номер
                └── Тариф
                └── Питание
                └── Источник / Агентство
```

### Гость (guest profile)

| Поле | Обязательность | Примечание |
|------|----------------|------------|
| ФИО | Да | Латиница + локаль |
| Документ (тип, №, срок) | При check-in (политика) | AZ: серия/Fin |
| Гражданство | Да для отчётов | |
| Телефон, email | Рекомендуется | SMS, счёт |
| Дата рождения | Опционально | CRM аналитика |
| VIP / теги | Нет | Из CRM |
| История визитов | — | Прошлые брони |

**Экраны:** Гостевые карточки WA0173, база гостей WA0243 (CRM).

### Бронь (reservation)

| Поле | Описание |
|------|----------|
| ID / внешний ID OTA | Связь с channel |
| Даты заезд–выезд | Ночи = выезд − заезд |
| Тип номера (requested) | Может отличаться от назначенного |
| Физ. номер | Пусто до assign |
| Тариф, питание | Снимок на момент брони |
| Adults / children | PAX |
| Статус | См. §4.2 |
| Источник, агентство | Комиссия, city ledger |
| Комментарии | Внутренние + guest requests |
| Гарантия | Deposit / card / agency |

### Проживание (in-house stay)

Создаётся при **check-in**. Связывает бронь + номер + открытые фолио.

## 4.2 Жизненный цикл брони

| Статус | Переходы | Кто |
|--------|----------|-----|
| **Option** | → Confirmed, Cancelled | Ресепшен / авто TTL |
| **Confirmed** | → In-house, No-show, Cancelled | Check-in / night audit |
| **In-house** | → Checked-out | Check-out |
| **Checked-out** | — | Финал |
| **Cancelled** | — | С penalty или без |
| **No-show** | Из Confirmed после даты | Night audit / вручную |

### Правила отмены

- По тарифу: бесплатно до N дней, иначе 1 ночь / %.  
- Штраф → charge на фолио или invoice agency.  
- Освобождение квоты → push channel.

### No-show

- Не заехал в день arrival без отмены.  
- Квота возвращается (политика).  
- Возможен штраф по тарифу.

## 4.4 Создание и редактирование брони

### Каналы создания

| Канал | Особенности |
|-------|-------------|
| Телефон / walk-in | Полный ввод |
| OTA import | Гость, цена, комиссия frozen |
| Корпорат / agency | City ledger flag |
| Группа | Block WA0106 |

### Обязательный минимум при создании

1. Даты  
2. Тип номера (или «не назначен»)  
3. Тариф + питание  
4. Гость или placeholder  
5. Источник  

### Редактирование в stay

| Изменение | Ограничение |
|-----------|-------------|
| Продление дат | Проверка availability |
| Сокращение | Пересчёт charges (политика) |
| Смена тарифа | Только manager+ ; audit log |
| Смена питания | С сегодня / со всей stay |

**Журнал изменений:** WA0192 (night audit module).

## 4.5 Room Plan (шахматка)

**Экран:** WA0209 (!Room Plan!).

| Элемент | Поведение |
|---------|-----------|
| Ось Y | Физические номера (сортировка этаж) |
| Ось X | Время (дни, часы опционально) |
| Блок брони | Цвет по статусу; подпись гость/агентство |
| Drag | Move dates / room (права) |
| Double-click | Карточка брони |

**Правила:**

- Нельзя пересечь два in-house на одном номере.  
- OOO номер — визуально серый, нельзя assign.

**Реализация (era-hotel-pms):** [`/room-plan`](../../app/room-plan/page.tsx) — Gantt 14/30 дней, `GET /api/room-plan`, продление `PATCH /api/reservations/:id/schedule` (+1/+2 nights). Drag-resize — фаза 1.1.

### Номерной фонд (Room Rack)

**Экран:** WA0206.

Карточки на сегодня: номер, иконка HK, гость, баланс (ссылка на фолио), agency.

## 4.6 Назначение номера и заезд

### Быстрое назначение номера

**Экран:** WA0184.

| Зона UI | Назначение |
|---------|------------|
| Список «не распределены» | Arrival today + confirmed |
| Список номеров | Подходящие по типу |
| Действие Assign | Связать бронь ↔ номер |

**Массовое:** выделить несколько → auto-assign по этажу/типу (алгоритм фаза 1.1).

### Check-in (процесс)

| Шаг | Проверка | Результат |
|-----|----------|-----------|
| 1 | Бронь confirmed, arrival ≤ today | |
| 2 | Номер assigned, статус HK не OOO | |
| 3 | Документ гостя (если политика) | |
| 4 | Мед. список (санаторий) | WA0126 |
| 5 | Открыть folio | Charge возможен |
| 6 | Статус → In-house | Room rack обновлён |

**Early check-in:** до 14:00 — доп. charge early arrival (код из WA0086).

### Check-out (процесс)

| Шаг | Проверка |
|-----|----------|
| 1 | Баланс folio = 0 или разрешён credit (agency) |
| 2 | Мини-бар / extras проверены |
| 3 | Invoice выписан (если нужно) |
| 4 | Статус → Checked-out |
| 5 | HK → dirty (авто) |

## 4.14 Бизнес-правила (свод)

1. Одна активная in-house бронь на физический номер.  
2. OOO не assignable, не sellable.  
3. OTA бронь: цена import locked unless manager override.  
4. Night audit не переводит дату, если есть незакрытые folio с policy block.  
5. Дети: учёт в PAX и тарифе (детская кровать).  
6. Medical tariff: может включать процедуры (связь MED — будущее).

## 4.15 Исключения

| Ситуация | Обработка |
|----------|-----------|
| Walk-in без брони | Создать бронь + check-in same flow |
| Upgrade при заезде | Room move + tariff diff charge |
| Share (2 гостя одна бронь) | 2 guest profiles, 1 folio или split |
| Day use | Бронь 0 nights, charges hourly — фаза 1.1 |
| Overbooking | Warning; manager release block or move |

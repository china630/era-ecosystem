# 6. Channel manager, CRM и санаторий (углублённо)

Три контура в одном документе — все относятся к гостю и выручке, но разные роли.

---

# Часть A — Channel Manager

## A.1 Назначение

Синхронизация **квот и цен** с OTA → входящие брони в PMS без двойного ввода.

## A.2 Сущности

| Сущность | Описание |
|----------|----------|
| Channel | Booking.com, Expedia, Exely… |
| Room type mapping | PMS STWN ↔ OTA "Standard Twin" |
| Rate mapping | BAR ↔ OTA rate plan |
| Restriction | Min stay, CTA, CTD |
| Sync log | Push/pull история |

## A.3 Настройка (setup)

| Экран | Функция | Ref |
|-------|---------|-----|
| Каналы OTA | Credentials, статус | WA0068, WA0250 |
| Online agency default | Дефолт для engine | WA0251 |
| ElektraSync | Параметры коннектора | WA0253 |
| Integration / iCal | Доп. каналы | WA0246 |
| Соглашения GDPR | Тексты для B2C | WA0255 |

## A.4 Операционная работа (ежедневно)

### Экстранет: цена и доступность

**Экран:** WA0127.

| Измерение | UI |
|-----------|-----|
| Даты | 7–30 дней вперёд |
| Типы номеров | Строки |
| Каналы | Колонки или filter |
| Ячейка | Price + rooms to sell |

**Действия:**

- Bulk update: +10% на weekend.  
- Stop sell (0 quota).  
- Push to channel.

### Мониторинг синхронизации

| Экран | Содержание | Ref |
|-------|------------|-----|
| Канал отправки данных | Очередь push | WA0247 |
| Журнал канала | Errors, last success | WA0248 |

**Типовые ошибки:** mapping missing, rate lock, connection timeout.

## A.5 Входящая бронь OTA

| Шаг | Система |
|-----|---------|
| 1 | Pull reservation XML/JSON |
| 2 | Match room type / rate |
| 3 | Create PMS reservation + guest |
| 4 | Confirm ack to OTA |
| 5 | Modify/cancel sync back |

**Правила:**

- Цена OTA frozen в брони.  
- Commission % в отчёте (не в folio guest).  
- Conflict: overbooking → alert manager.

## A.6 Пользовательские истории

| ID | История |
|----|---------|
| CH-01 | Revenue закрывает продажи на 1 января (0 quota) all channels |
| CH-02 | Ошибка mapping — менеджер видит в журнале и исправляет |
| CH-03 | OTA cancel → PMS бронь cancelled автоматически |

## C.6 Связь с тарифом Medical

**Nafta (рабочая гипотеза, 2026-05):** основные мед. услуги санатория **включены в оплату номера**; отдельные строки на folio — **исключения** (доп. процедуры, товары). Подтвердить список исключений на встрече — [13-nafta-validation-checklist.md](13-nafta-validation-checklist.md) D3.

| Фаза | Поведение в PMS |
|------|-----------------|
| Ф.1 | MED-04 и quick posting — начисление **по факту**; пакет «N процедур/день» в тарифе **не списывается** автоматически |
| Ф.2 | Учёт пакета Medical rate plan — остаток процедур, без дублирования charge за включённое |

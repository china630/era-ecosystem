# 17. Соответствие законодательству Азербайджана (фаза 2)

> Детальная аналитика: [../reference/elektraweb-gap-analysis-az-global.md](../reference/elektraweb-gap-analysis-az-global.md)  
> **Не копируем:** e-Fatura, KBS, Yazarkasa (Турция)

## 17.0 Принцип размещения функций

```
┌──────────────┐     ┌─────────────────┐     ┌──────────────────┐
│  Nafta PMS   │────▶│ Fiscal gateway  │────▶│ taxes.gov.az     │
│  (операции)  │     │ или ERP         │     │ e-qaimə / KKM    │
└──────────────┘     └─────────────────┘     └──────────────────┘
        │                     │
        └─────────────────────┴──▶ Tourism registry (data.tourism.gov.az)
```

PMS хранит **операционный факт** (счёт, платёж, гость).  
**Юридически значимая отправка** — через сертифицированный шлюз или ERP.

---

## 17.1 e-qaimə (электронная счёт-фактура)

| Аспект | Требование AZ | Роль PMS | Роль ERP/шлюз |
|--------|---------------|----------|---------------|
| VAT 18% | На B2B счетах | Метка на коде дохода | Расчёт строк |
| XML + ЭП | Asan İmza / e-Signer | — | Формирование и отправка |
| Хранение 5 лет | Архив | Копия metadata + ссылка | Оригинал |
| Статус | Accepted / rejected | **Read-only** в карточке invoice | Write |

### Триггеры из PMS

| Триггер | Данные для шлюза |
|---------|------------------|
| Invoice «выставлен» (B2B) | Контрагент VÖEN, строки, НДС |
| Credit note | Ссылка на исходный e-qaimə |
| Агентство city ledger | Периодический пакет (ERP) |

**Событие:** расширение **E2** полями `fiscal_status`, `fiscal_id` (см. [18-erp-integration.md](18-erp-integration.md)).

## 17.2 Онлайн-KKM (B2C, касса)

| Сценарий | Где фискализировать |
|----------|---------------------|
| Оплата на ресепшен (наличные/карта) | KKM ресепшен или ERP |
| POS-ресторан | KKM outlet |
| Quick posting на folio | Политика отеля: один KKM на объект или defer до check-out |

| Поле чека | Источник PMS |
|-----------|--------------|
| Сумма AZN | Payment |
| НДС | Revenue code |
| QR для покупателя | Ответ KKM API |

**Событие E7:** `payment_fiscalized` — id чека, QR payload, timestamp.

**Не делаем в PMS:** прямое подключение к ККМ без абстракции `FiscalProvider`.

## 17.3 Реестр туризма

Elektraweb TR: KBS WA0112 — **не применимо**.

| Данные | Когда отправлять |
|--------|------------------|
| Гость: ФИО, документ, даты | Check-in / check-out |
| Номер, тип размещения | Check-in |
| Отель: рег. номер | Справочник |

**Модуль PMS:** `TourismRegistryAdapter` — конфиг URL/credentials AZ.

**User story:**

| ID | История |
|----|---------|
| AZ-01 | Check-in → запись уходит в tourism.gov.az в течение N минут |
| AZ-02 | Ошибка реестра → ресепшен видит alert, повтор вручную |

## 17.4 Валюта и курсы

| Правило | Фаза |
|---------|------|
| Базовая валюта AZN | 1 |
| Курс EUR/USD на дату | 1 (ручной) |
| Курс ЦБ AZ auto | 2 |

ERP — учёт курсовых разниц; PMS — операционный курс дня.

## 17.6 VAT refund для туристов

Отраслевая практика AZ (возврат НДС).  
**Фаза 2+:** отдельный отчёт / интеграция с порталом — **не блокер go-live** санатория.

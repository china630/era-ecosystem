# 1. Граница финансов: ACC в Elektraweb vs наш продукт

## Короткий ответ

В Elektraweb модуль **ACC (Muhasebe)** по задумке вендора — это **и** отельная подсобная бухгалтерия, **и** задел под полноценный учёт (проводки, план счетов, aging).  

У **Nafta на скринах** видно другое: повседневные деньги идут через **«Касса» / фолио / ночной аудит**, а ACC в основном — **настройки, справочники и шлюзы выгрузки** (Sage, e-invoice, НДС, банки), а не журнал проводок.

**Для нашего PMS:** делаем **операционный финансовый контур** + **события для ERP**. Проводки и NAS — **не в фазе 1**.

## Три слоя (как разделить ответственность)

```
┌─────────────────────────────────────────────────────────────┐
│  Слой A — Операции отеля (наш PMS, фаза 1)                 │
│  Бронь → проживание → начисления на фолио → оплата →        │
│  закрытие операционного дня (night audit)                   │
└──────────────────────────┬──────────────────────────────────┘
                           │ события / пакеты данных
┌──────────────────────────▼──────────────────────────────────┐
│  Слой B — Внешнее ERP Nafta (уже есть)                       │
│  Проводки, GL, NAS, дебиторка/кредиторка, закрытие месяца   │
└──────────────────────────┬──────────────────────────────────┘
                           │ при необходимости
┌──────────────────────────▼──────────────────────────────────┐
│  Слой C — Гос. / fiscal AZ (позже)                          │
│  e-qaimə, KKM, tourism registry                             │
└─────────────────────────────────────────────────────────────┘
```

## Что есть в Elektraweb ACC (по маркетингу)

Полный ERP-модуль: план счетов, **бухгалтерские фиши (Muhasebe Fişleri)**, платёжные поручения, aging, фин. отчёты, курсовые разницы, центры затрат.

## Что видно у Nafta в ACC (по 292 скринам)

| Тип | Примеры | Суть |
|-----|---------|------|
| **Маппинг на будущий учёт** | Коды дохода (ROOM, MEDICAL, SPA STORE), отделы, настройки бухгалтерии WA0097 | Куда в учёте «лягут» операции, не сами проводки |
| **Интеграции / выгрузка** | Экспорт Sage WA0274, Bavel, настройки e-invoice WA0278, WA0304 | Передача **уже сформированных** документов во внешние системы |
| **Справочники для ERP** | НДС, банковские счета, курсы WA0272 | Референс для счетов-фактур и интеграций |
| **Импорт master data** | Панель импорта WA0298, продукты, рецепты | Массовая загрузка номенклатуры (скорее STOCK+ACC, не GL) |
| **Правила склада/закупок** | WA0321 | Бизнес-правила, не проводки |

**Не найдено** в выгрузке Nafta: экраны ручного ввода проводок, оборотно-сальдовая, журнал GL, платёжные ведомости.

## Что в Elektraweb делает «Касса» (CASH) — это наш главный финансовый UX

| Функция | Назначение |
|---------|------------|
| Список счетов-фактур | Операционные инвойсы гостям/компаниям |
| Кассовые операции | Платежи, возвраты, привязка к фолио |
| Предоплата | Депозиты до заезда |
| Folio routing | Разделение: компания / гость |
| City ledger / Agency statement | Долги агентств (B2B), не GL |
| Курсы валют | Операционный курс дня |
| Quick posting | Быстрое начисление услуги на номер |

Это **не проводки** — это **транзакции гостиницы**, которые потом можно агрегировать в ERP.

## Night audit — операционное закрытие, не месяц бухгалтерии

- Перенос тарифов проживания на счета гостей  
- Проверка незакрытых касс  
- Смена операционной даты  
- Архив Z-отчётов / сводок дня  

Аналог **операционного** cut-off, не «закрытие месяца» в GL.

## Решение для Nafta PMS

### Внутри PMS (фаза 1)

| Сущность / действие | Описание |
|---------------------|----------|
| **Код дохода** | Тип услуги для аналитики (номер, питание, мед., SPA…) |
| **Отдел (department)** | Кто «заработал» выручку (проживание, ресторан, мед.) |
| **Начисление (charge)** | Сумма на фолио гостя |
| **Платёж (payment)** | Наличные, карта, банк, аванс |
| **Счёт (invoice)** | Документ для гостя/компании (операционный) |
| **City ledger** | Долг агентства, сверка — без GL |
| **Кассовая смена** | Открытие/закрытие, контроль наличности |
| **Night audit** | Закрытие операционного дня |

### Снаружи PMS (ERP Nafta, позже)

| Событие для ERP | Когда |
|-----------------|-------|
| Дневной пакет выручки по кодам дохода / отделам | После night audit |
| Выписанные счета-фактуры (шапка + строки) | По факту выписки |
| Платежи и депозиты | По факту кассы |
| Сальдо city ledger по агентствам | По запросу / период |

Формат интеграции (API, файл) — **вне этой спецификации**; здесь только **какие бизнес-факты** отдаём.

### Сознательно не делаем в PMS

- План счетов NAS, синтетика/аналитика  
- Ручные и автоматические **бухгалтерские проводки**  
- Закрытие месяца, баланс, ОСВ  
- Зарплата, закупки, амортизация ОС  
- Турецкий e-Fatura / KBS как эталон

## Сравнение с модулем ACC в лицензии Elektraweb

| Вопрос | Elektraweb ACC (лицензия) | Nafta PMS (наш план) |
|--------|---------------------------|----------------------|
| Проводки? | Заявлены в продукте | **Нет** — в ERP |
| Операции для 1C/ERP? | Частично (Sage, e-invoice) | **Да** — основной мост |
| Касса и фолио? | Отдельный контур CASH | **Да** — ядро |
| НДС / счета AZ? | TR-интеграторы | **Позже** на стыке AZ |

**Вывод:** ваше понимание верное — начинаем с **сугубо отельных операций**; ACC Elektraweb для Nafta по факту ближе к **«настройка + выгрузка»**, а не к ежедневной бухгалтерии в PMS.

## 15.0 Обзор

| Контур | Назначение | Экраны Nafta |
|--------|------------|--------------|
| **POS-зал** | Заказы в ресторане, столы, смена кассы | *не сфотографированы* |
| **POS-календарь** | Бронь стола / SPA-слота | WA0146 |
| **Quick posting** | Пробить услугу на folio без зала | WA0135 (ф.1) |
| **SPA-касса** | Продажа процедур, мастер, расходники | *не сфотографированы* |
| **Настройки POS** | Чек, X/Z, лимиты | WA0098 |

**Решение Nafta (из чеклиста):** если ресторан = quick posting + kitchen вне PMS → пакет **15-lite** без карты зала.

**Связанные продукты:**

| Продукт | Назначение |
|---------|------------|
| `era-hotel-pms` | Quick posting (WA0135), мост room-charge, календарь слотов, медконтур |
| `era-fb-pos` | Полный F&B POS (Food & Beverage) — зал, KDS; **не** розничный магазин |
| `era-spa-pos` | **Не планируется** для Nafta без нового scope; SPA — quick posting + мед + календарь в PMS |

Архитектура PMS ↔ fb-pos ↔ ERP: [23-pos-bridge.md](23-pos-bridge.md), [13-nafta-validation-checklist.md](13-nafta-validation-checklist.md) §H.

## 15.2 POS-зал (полный контур)

### 15.2.1 Карта зала

| Статус стола | Цвет / правило |
|--------------|----------------|
| Free | Принять гостей |
| Occupied | Есть открытый ticket |
| Reserved | Бронь WA0146 |
| Dirty | После закрытия (опционально) |

**Действия:** открыть чек, перенести стол, объединить столы, split.

### 15.2.2 Заказ (официант)

| Шаг | Действие |
|-----|----------|
| 1 | Выбор outlet + стол (или walk-in) |
| 2 | Добавление позиций из меню (категории, поиск, штрихкод) |
| 3 | Модификаторы, комментарий кухне |
| 4 | Hold / fire (если KDS) |
| 5 | Пречек / оплата / transfer to room |

**Связь с гостем:** поиск in-house по номеру / фамилии → folio id.

### 15.2.3 KDS (кухня)

| Поле тикета | Описание |
|-------------|----------|
| Table / guest | |
| Items | Статус: new / in prep / done |
| Timer | SLA кухни |
| Reprint | Повтор печати |

*В Elektraweb на скринах Nafta не подтверждён — флаг «опционально».*

### 15.2.4 Оплата и закрытие чека

| Способ | Учёт |
|--------|------|
| Наличные / карта | POS shift + событие E3 |
| **На номер (room charge)** | Charge на folio, код FOOD/SPA |
| Split bill | Доля по гостям / способам |
| Discount | Права manager, причина |

**Правило:** room charge только для **in-house** с открытым folio.

### 15.2.5 Смена POS (X / Z)

По WA0098:

| Настройка | Смысл |
|-----------|--------|
| Контроль открытых столов | Блок Z-report если есть open tickets |
| X-отчёт | Промежуточный без закрытия |
| Z-отчёт | Закрытие смены POS |
| Комиссия % | Для эквайринга |

**Связь с night audit:** открытая POS-смена может блокировать закрытие дня (как касса ф.1).

## 15.3 POS-календарь (бронь ресурса)

**Экран:** WA0146.

| Поле | Описание |
|------|----------|
| Resource | Стол / SPA-кабинет |
| Date / time slot | Сетка 00:00–24:00 |
| Guest / reservation link | Опционально |
| Complex | Restaurant vs SPA facility |

**User stories (фаза 2):**

| ID | История |
|----|---------|
| POS-01 | Хостес бронирует стол 19:00 на 4 персоны |
| POS-02 | Администратор SPA бронирует кабинет + процедуру |
| POS-03 | При заезде гостя видна бронь стола в календаре |

## 15.4 SPA-касса

Отдельный UX от мед. модуля ([06-channel-crm-med.md](06-channel-crm-med.md)):

| Функция | Мед. модуль (ф.1) | SPA-касса (ф.2) |
|---------|-------------------|-----------------|
| Мед. список дня | ✅ | — |
| Назначения врача | ✅ | — |

| Расходники | — | Списание со склада (16) |
| Комиссия мастера | — | Отчёт (опционально) |

**Справочник услуг:** пересечение с WA0282 (маппинг → код MEDICAL / SPA STORE).

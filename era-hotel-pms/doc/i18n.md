# UI internationalization (i18n)

Stack: [next-intl](https://next-intl.dev) · locales **en**, **ru**, **az** (aligned with [09-master-data.md](clone-spec/09-master-data.md) RU/AZ/EN labels).

## Behaviour

- Language is stored in cookie `NEXT_LOCALE` (no `/ru/` prefix in URLs).
- Switcher: top-right on **Login** and **AppNav** (all authenticated pages).
- Default: `DEFAULT_LOCALE` in `.env` (fallback `en`).

## Adding strings

1. Add keys to `messages/en.json`, then `ru.json` and `az.json`.
2. Client components: `const t = useTranslations('namespace');` → `t('key')`.
3. Server components / metadata: `getTranslations('namespace')` from `next-intl/server`.

## Coverage

All routes under `app/**/page.tsx` use message files. API error strings from the server may still be in English.

New UI work: add keys to `messages/en.json`, `ru.json`, and `az.json`, then `useTranslations('namespace')`.

## API

`POST /api/locale` `{ "locale": "ru" }` — public (works before login).
